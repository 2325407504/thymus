Thymus
tar.gz .zip

thymus.js is a small (1k minified) JavaScript template library for including/substituting client side fragments using HTML5 resources that reside in different paths on a server and/or file system.

Internally, it dynamically loads the JQuery API (configurable) and uses JQuery $.get(...) calls to retrieve and insert fragments into pages. The process is similar to how JQuery $.load(...) works with page fragments with a few added benefits. thymus.js will automatically insert fragments into a page using HTML5 data attributes within the page and will update any relative src/href URLs within fragments to point to the proper path before insertion. It also supports nested fragments that may reside in other fragments. Click here for a full list of features.

This site actually uses thymus.js templates to resolve fragments for the top navigation bar, bottom navigation bar, example user profile drop down menu, page head and scripts. So, feel free to explore :)

Requirements
  • JQuery >= 1.9.1

Features

  • Automatic fragment inclusion
  • Automatic fragment substitution
  • Automatic fragment src/href resolution for relative paths
  • Multiple fragment definitions in the same fragment resource
  • Relatively infinite nesting of fragments (fragments within fragments)
  • Head tag fragments
  • Fragment loaded event capture/execution
  • Great for prototyping web sites on the client!

Basic Usage

To use thymus.js it obviously needs to be included in the page. Here is the minimal setup: 

<head>
<meta charset="utf-8" />
<!-- Thymus script -->
<script src="js/thymus.js" id="thymusScript" data-th-context-path="."></script>
</head>

There are two required attributes for thymus.js. The thymusScript id is used to capture various attribute options on the script tag while the data-th-context-path is used for fragment path resolution. 

<head data-substitue="frags/head :: header">
<meta charset="utf-8" />
<!-- Thymus script -->
<script src="js/thymus.js" id="thymusScript"
	data-th-context-path="."
	data-th-head-frag="data-substitue=frags/head :: header"></script>
</head>

Fragment Path Resolution

Because thymus.js runs on the client it's not inheriently application context aware. Therefore, thymus.js uses a configurable data-th-context-path attribute on the thymus.js script tag. It's value establishes the path to the root context of the application that will be resolved along with the fragment's inclusion URL. Fragment definitions can point to resources with different path levels. So, it's imporant that the data-th-context-path synchronizes with the path defined within the fragment. For example, a fragment that resides in example.com/my-app/frags/nav.htm may be included in example.com/my-app/index.htm as well as example.com/my-app/user/info.htm. If the fragment is defined something like: 

<div data-th-fragment="topnav">
	...
</div>

Then both example.com/my-app/index.htm and example.com/my-app/user/info.htm could both contain a corresponding include (or substitution) that has the same fragment path: 

<div data-th-include="frags/nav/nav :: topnav">
	...
</div>

This vasly simplifies fragment path resolution (escpecially for nested fragments) and ensures that thymus.js uses the proper path. In the case of example.com/my-app/index.htm and example.com/my-app/user/info.htm where data-th-context-path="." the path within data-th-include="frags/nav/nav :: topnav" would get resolved to example.com/my-app/ example.com/my-app/user/info.htm 

<head data-substitue="frags/head :: header">
<meta charset="utf-8" />
<!-- Thymus script -->
<script src="js/thymus.js" id="thymusScript"
	data-th-context-path="."
	data-th-head-frag="data-substitue=frags/head :: header"></script>
</head>

While info.htm is within a sub-directory of the root directory called user. So, we could define data-th-context-path within info.htm as: 

<head data-substitue="frags/head :: header">
<meta charset="utf-8" />
<!-- Thymus script -->
<script src="js/thymus.js" id="thymusScript"
	data-th-context-path="."
	data-th-head-frag="data-substitue=frags/head :: header"></script>
</head>

Also, URLs contained within nav.htm can contain relative paths within src and href attributes. thymus.js will automatically resolve any relative paths to absolute paths relative to the data-th-context-path. 

<head data-substitue="frags/head :: header">
<meta charset="utf-8" />
<!-- Thymus script -->
<script src="js/thymus.js" id="thymusScript"
	data-th-context-path="."
	data-th-head-frag="data-substitue=frags/head :: header"></script>
</head>

Thymeleaf Integration

Thymeleaf is a great server-side template engine for Java and is the inspiration behind Thymus. In fact, by default Thymus uses the same default HTML5 attributes for includes/substitutions and fragment definitions. 

<head th:substituteby="frags/head :: header">
<meta charset="utf-8" />
<!-- Thymus script -->
<script th:remove="all" src="js/thymus.js" id="thymusScript" 
	data-th-context-path="." data-th-frag-extension=".htm"
	data-th-head-frag="th:substituteby=frags/head :: header"
	data-th-onfragsloaded="init(this)"></script>
</head>
