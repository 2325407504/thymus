thymus.js

Internally, it dynamically loads the JQuery API (configurable) and uses JQuery $.ajax(...) calls to retrieve and insert fragments into pages. The process is similar to how JQuery $.load(...) works with page fragments with a few added benefits. thymus.js will automatically insert fragments into a page using HTML5 data attributes and will update any relative src / href URLs within returned fragments to point to the proper path before insertion. It also supports nested fragments that may reside in other fragments. Click here for a full list of features.

This site actually uses 0 thymus.js fragments to while initializing this page. So, feel free to explore :)

Features
  • Automatic fragment inclusion while a page loads (configurable)
  • Automatic fragment replacement while a page loads (configurable)
  • Automatic fragment src/href resolution for relative paths (configurable)
  • Load fragments via script using JQuery selectors
  • Multiple fragment definitions at the same fragment URL
  • Prevents repetitive calls to the same URL associated with raw JQuery.load content (typical case when multiple fragments reside at the same URL)
  • Self-Referencing templates
  • Relatively infinite nesting of fragments (fragments within fragments)
  • head tag fragments
  • Ability to override the processing of fargment results via function
  • Automatic fragment loading or deferred fragment loading
  • Maintains vertical scrolling positions associated with named anchors that point to a location within a fragment (e.g. example.com/path#someNamedAnchorInAFrag)
  • Fragment events dispatched for each resource has been loaded or failed to load (including scripts and nested fragments)
  • Fragment event dispatched when all fragments have been loaded or failed to load (including scripts and nested fragments)
  • Complete JQuery plug-in extensibility
